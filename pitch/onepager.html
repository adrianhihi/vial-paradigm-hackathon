What Vial is. A domain-agnostic runtime that watches an agent or automation execute, detects failures, and repairs them in-place using learned strategies persisted in a Gene Map. 553 tests, v2.7.0, open-source core at @vial-agent/runtime.

Why it matters for automated research. Automated research is a PCEC loop: propose a solution, evaluate it, diagnose why it failed, try again with what you learned. Vial already implements this loop as runtime primitives — we just needed to point it at a new domain.

① The PCEC Loop
P
Perceive
Score below baseline, plateau, compile error, routing anomaly
C
Construct
Retrieve relevant capsules from Gene Map, condition next generation
E
Evaluate
Run on local Paradigm harness, measure Edge, parse failure signals
C
Commit
If improved, synthesize new capsule; if regressed, decay confidence
Every capsule is human-readable — trigger category, repair strategy, rationale. Not opaque weights. Auditable, explainable, portable across runs.
